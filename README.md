# Renamer

다운로드 폴더에 새 PDF 또는 이미지 파일이 추가되면 LLM이 그 내용을 분석하여 자동으로 파일 이름을 바꿔주는 프로그램입니다.

- macOS 전용 앱을 먼저 만들고, 그것과 최대한 같은 기능 및 UI/UX를 Windows에서 구현하고자 한 것입니다. macOS 앱에 비해 불완전하거나, 혹은 추후 업데이트가 느릴 수 있습니다.
- 본 앱은 AI를 통해 PDF 및 이미지 파일을 분석합니다. 현재 연결 가능한 LLM은 Claude입니다.
- 즉, 본 앱을 사용하기 위해서는 Claude API 이용이 필요합니다. (이에 관해서는 아래의 ‘5. Claude API 설정 방법’을 참고)

## 1. 핵심 기능

### 1.1. 논문(PDF 파일)의 파일 이름 자동 수정

다운로드 폴더에 PDF가 추가되면 해당 PDF 파일의 처음과 끝 부분을 확인하여 학술지 논문(Journal Article)에 해당하는지 확인하고, 해당될 경우 (1) **논문 제목**(본제목) (2) **저자 이름**(성+이름) (3) **발행 연도** 정보를 추출한 뒤 이를 조합하여 파일 이름을 자동으로 수정해줍니다.

저자가 2명 이상인 경우 다음과 같은 규칙을 따릅니다.
- 저자가 2명일 때에는, 한국어인 경우 가운뎃점(·)으로, 영어인 경우 앤퍼센트(&)로 구분
- 저자가 3명 이상일 때에는 대표저자 1명만 명시하고, 한국어인 경우 ‘외’, 영어인 경우 ‘et al.’ 표기

```
KCI_FI003340714.PDF → 정재훈(2026), 검열, 프로파간다, 영화적 다중첩자.pdf
KCI_FI002974382.pdf → 유승진·백문임(2023), ‘전후(戰後)‘의 영화 미학과 정동의 공동체.pdf
KCI_FI003327008.pdf → 백문임 외(2026), ‘부활’한 다큐멘터리.pdf
```

> 윈도우용 앱의 경우, 디지털 PDF 또는 이미 OCR 처리가 된 PDF 파일은 문제 없이 처리하지만, OCR 처리가 안 된 스캔본 PDF는 읽어내지 못합니다. 이는 맥용 앱의 경우 애플 기기에서 작동하는 Apple Vision 프레임워크를 사용하여 이미지에서 즉시 텍스트를 추출할 수 있게 되어 있지만, 윈도우용 앱에서는 이러한 기능이 없기 때문입니다. (~~억울하면 맥을 삽시다.~~)

### 1.2. 학술 행사 포스터(이미지 파일)의 파일 이름 자동 수정

다운로드 폴더에 JPG, PNG 등 이미지 파일이 추가되면 (학술) 행사 포스터인지 여부를 판단하고, 포스터에 해당될 경우 (1) **행사 제목** (2) **날짜**(연·월·일) 정보를 추출한 뒤 이를 조합하여 파일 이름을 자동으로 수정해줍니다.

```
1769761623137.jpeg  →  (포스터)상허학회 2026년 겨울 정기학술대회(260220).jpeg
```

주의: 이미지 파일의 인식은 논문 PDF 파일의 인식에 비해 좀 더 오류가 많을 수 있습니다.

### 2.1. Claude 모델 선택

- Claude에서 어떤 모델을 적용할지 선택할 수 있습니다. 그렇지만 결론부터 말하면, **되도록이면 Haiku 모델의 사용을 권합니다.**
- 선택지는 (2026년 6월 기준) Haiku 4.5와 Sonnet 4.6 이렇게 2가지 입니다.
    - Haiku는 성능이 뛰어나지 않은 대신 가볍고 빠르다는 장점이 있습니다. 본 앱의 작동을 위해서는 높은 성능이 요구되지 않으므로, 대부분의 경우 Haiku 모델로도 충분합니다.
    - Sonnet는 Haiku보다 성능이 뛰어나므로, 그만큼 PDF 및 이미지 파일에서 정보를 읽어들일 때 오류 확률이 줄어들 수 있습니다. 그 대신 Haiku에 비해 토큰(AI 사용량을 따지는 단위)당 가격이 더 비쌉니다(3배).
    - Claude에는 가장 성능이 뛰어난 모델인 Opus가 있습니다. 단, 본 앱에서는 이 정도의 성능은 필요하지 않고 토큰당 비용만 높을 뿐이라고 여겨 선택지로는 제공하지 않기로 했습니다. 

### 2.2. 적용 대상 파일 종류 선택

- 분석 및 파일 이름 자동 수정 기능의 적용 대상이 되는 파일 종류(PDF/이미지)를 선택할 수 있습니다.
- **off할 경우 파일 이름 수정을 하지 않을 뿐 아니라 분석 대상에서 제외되므로 Claude API 사용으로 인한 비용 발생을 차단할 수 있습니다.**

### 2.3. 파일 이름 규칙 커스터마이징

- 디폴트로 설정된 파일 이름 형식은 예컨대 다음과 같습니다.

```
김연구(2026), 읽어야 할 논문은 왜 이토록 많은가.pdf
(포스터)편리한 연구를 위한 좌담회(260601).jpeg
```

- 변수 개념만 이해하면 표현식 수정을 통해 파일 이름 형식을 마음대로 설정할 수 있습니다.

#### 2.3.1. 논문 PDF 파일의 경우
- PDF 파일의 일부를 AI가 읽고 추출하는 정보는 다음 세 가지입니다: (1) **논문 제목**(본제목) (2) **저자 이름**(성+이름) (3) **발행 연도**
- 이는 각각 다음과 같은 변수에 저장됩니다: (1) `{name}` (2) `{title}` (3) `{year}`
- 즉, 위의 예시의 경우 각 변수에 저장된 정보값은 다음과 같습니다: (1) `{name} = "김연구"` (2) `{title} = "읽어야 할 논문은 왜 이토로 많은가"` (3) `{year} = "2026"`
- 디폴트 파일 이름 형식은 다음과 같이 표현됩니다(여기의 각 변수 부분에 값을 집어넣으면 결과물이 나오는 것입니다):

```
{name}({year}), {title}
```

- 표현식에서 중괄호(`{}`)로 감싸진 것이 아닌 부분은 바뀌지 않고 파일 이름마다 동일하게 들어가는 문자열이 됩니다. 
- 예를 들어, ‘김연구, 「읽어야 할 논문은 왜 이토록 많은가」(2026)’와 같은 형식으로 바꾸고 싶을 경우, 표현식을 다음과 같이 쓰면 됩니다:

```
{name}, 「{title}」({year})
```

#### 2.3.2. 학술 행사 포스터 이미지 파일의 경우
- 이미지 파일의 일부를 AI가 읽고 추출하는 정보는 크게 다음 두 가지입니다: (1) **행사 제목** (2) **날짜**(연·월·일)
- 이는 각각 다음과 같은 변수에 저장됩니다: (1) `{title}` (2) `{when: YYMMDD}`
- 즉, 위의 예시의 경우 각 변수에 저장된 정보값은 다음과 같습니다: (1) `{title} = "편리한 연구를 위한 좌담회"` (2) `{when: YYMMDD} = 2026년 6월 1일`
- 일시 변수 즉 `{when: YYDDMM}`에서 괄호 내부, 콜론(:) 다음에 나오는 날짜 표현 형식을 바꿀 수 있습니다.
    - `YY`: 4자리의 서력기원 연도 중 마지막 두 자리만 나타냅니다.
    - `YYYY`: 4자리의 서력기원 연도 전체를 나타냅니다.
    - `M`: 1월부터 9월까지는 한 자리 숫자로, 10월부터 12월까지는 두 자리 숫자로 월을 나타냅니다.
    - `MM`: 1월부터 9월까지는 앞에 숫자 0을 붙여 나타내고(예: 06월), 10월부터 12월까지는 그대로 나타냅니다.
    - `D`: 1일부터 9일까지는 한 자리 숫자로, 10일부터 31일까지는 두 자리 숫자로 일을 나타냅니다.
    - `DD`: 1일부터 9일까지는 앞에 숫자 0을 붙여 나타내고(예: 01일), 10일부터 31월까지는 그대로 나타냅니다.
    
|표현식|결과 예시|
|----|-------|
|`YYYY. M. D.`|2026. 6. 1.|
|`YY/MM/DD`|26/06/01|
|`YY년 M월 D일`|26년 6월 1일|

#### 2.4. 로그 설정 및 초기화

- 최대 몇 건까지 로그를 저장하고 보여줄지 선택할 수 있습니다. (100 / 250 / 500 / 1000건)
- 로그를 초기화, 즉 전체 삭제할 수 있습니다.

## 3. 통계

<p align="center"><img src="https://github.com/user-attachments/assets/2a34e26e-c62f-417c-a591-18e5cb8c5db0" width="360"></p>

- ‘설정’ 버튼과 ‘종료’ 버튼 가운데에 있는 ‘통계’를 클릭하면 팝오버가 뜹니다.
- 여기서 다음 세 가지 수치를 확인할 수 있습니다.
    1. 분석 건수: 논문인지 여부를 확인한 모든 PDF 파일의 수 + 포스터인지 여부를 확인한 모든 이미지 파일의 수
    2. 파일 이름 수정 건수: 논문/포스터 여부를 확인한 파일 중 이름을 수정한 파일의 수
    3. 누적 API 비용: 분석(Input) 및 파일 이름 수정(Output) 과정에서 입력 및 출력된 토큰수와 사용한 모델 종류를 통해 계산된, Claude API 사용료 누적액 
- ‘통계 초기화’ 버튼을 클릭하여 기준 시점을 현재로 하여 카운트를 0부터 다시 시작할 수 있습니다.
- 하단의 ‘지난 30일 간’의 수치는 최근 한 달 간의 분석 건수, 파일 이름 수정 건수, 누적 API 비용을 나타냅니다. 이 수치는 위의 ‘통계 초기화’ 버튼의 영향을 받지 않습니다.

## 4. 설치 방법

1. [Releases](../../releases) 페이지에서 최신 `Renamer.zip` 다운로드
2. 압축 해제 후 원하는 위치에 폴더를 두고 `Renamer.exe` 실행
3. **처음 실행 시 Windows SmartScreen 경고가 뜰 수 있습니다**: ‘추가 정보’ 클릭 → **‘실행’** 클릭
4. 시스템 트레이(우하단)에 아이콘이 생겼는지 확인합니다.

> 코드 서명이 없어 SmartScreen 경고가 뜹니다. ‘추가 정보 → 실행’으로 한 번만 허용하면 이후엔 정상 실행됩니다.

## 5. Claude API 설정 방법

1. [Claude Console](https://platform.claude.com/dashboard)로 접속하여 가입 및 로그인합니다.
2. Anthropic사의 정책에 따라, Claude API의 사용을 위해서는 최초에 적어도 한 번은 크레딧 구매가 필요합니다. 최소 5달러(USD)부터 결제 가능합니다. 다행히도, 최초의 5달러 결제만 해두면 앞으로 추가 결제 없이 아주 오랫동안 Renamer를 사용할 수 있습니다. **[주의!]** 예상치 못한 청구서 폭탄이 날아오는 사태를 방지하기 위해, 크레딧 부족 시 자동 충전 설정을 끄거나 자동 충전 금액을 소액으로 낮춰놓는 것을 권장합니다.
3. 좌측 사이드바에서 ‘API 키’를 클릭한 뒤, 우측 상단의 ‘+ 키 생성’ 버튼을 클릭합니다.
4. 적당히 알아볼 수 있을 만한 이름을 입력한 뒤 ‘추가’를 클릭하고 ‘키 복사’를 클릭합니다.
5. Renamer를 실행하여 설정 창을 엽니다. ‘Claude API Key 입력하기’ 항목의 입력란에 복사해둔 Key를 붙여넣고 ‘검증 및 저장’ 버튼을 클릭합니다.
6. ‘✓ 저장됨’ 안내 문구가 뜨면 성공!

## 6. Claude API 사용에 따른 비용(예측치)

- 앱 사용 시 비용이 발생합니다(선불). 이는 이 앱의 개발자인 제가 아니라 Claude API 사용 대가로서 Anthropic사가 청구하는 비용입니다. 
- 결론부터 말하면, 논문 PDF 파일의 경우 건당 약 0.0015달러 내외, 행사 포스터 이미지 파일의 경우 건당 약 0.0035달러 내외의 비용이 발생합니다(Haiku 모델 사용 시). 각각 100건이 누적되어도 200원, 500원 꼴입니다(2026년 6월, 1달러 당 약 1500원의 환율 기준). 비싸다면 비싸고, 쓸데없는 시간과 번거로움을 줄이는 값으로 싸다면 싼 가격입니다. 판단은 사용자의 몫입니다.
- 더 정확하게 말하면, PDF 파일 혹은 이미지 파일이 다운로드 폴더에 추가될 때 일단 그것이 논문인지 아닌지를 파악하기 위해 AI에게 정보를 넘길 수밖에 없고, 결과적으로 논문이나 행사 포스터가 아닌 것으로 판단될 경우에도 Claude API의 Input 토큰은 소모하게 됩니다. 그러므로 API Key 입력이 되어 앱이 정상작동 중이면 파일 이름 자동 수정이 적용되든 안 되든, PDF 또는 이미지를 다운로드할 때마다 Input 토큰 만큼의 비용이 발생하게 됩니다.
- 이미지의 경우 AI가 이미지를 파악하게 하는 과정에서 매겨지게 되는 Input 토큰이 많은 편이고, 그로 인해 PDF보다 건당 비용이 많이 발생합니다. 평소에 이미지를 많이 다운로드하시는 분의 경우 또는 특정 시간 동안 여러 이미지를 다운로드하게 될 경우, 설정에 가서 포스터 이미지를 대상에서 제외(토글 스위치 off)해두는 것을 권합니다.
- Sonnet 모델은 같은 양을 사용할 시 발생하는 비용이 Haiku에 비해 **3배**나 됩니다. 되도록이면 Haiku 모델을 사용하는 것을 권장합니다.
